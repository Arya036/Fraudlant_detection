## Test Case 1: Live Stream + Live Alert Trigger

### Goal
Demonstrate continuous incoming transactions, live scoring, and real-time alert-object creation.

### Steps
1. On Command Center, set rate to `5 tx/sec`.
2. Keep `Live alerts` checkbox ON.
3. Click `Start Replay`.
4. Let it run for 20-30 seconds.

### Pass Criteria
1. `Scored` count increases continuously.
2. `Fraud flagged` count increases.
3. `Live alerts` count becomes greater than 0.
4. Live feed shows normal transaction rows plus at least one `LIVE ALERT` row.

### Evidence to mention
- Replay endpoint: `POST /api/simulate/start`
- WebSocket stream: `/ws/live-feed`
- Live alert object created and stored in `alerts` table during replay.

## Test Case 2: Manual Scoring + Explainability

### Goal
Show model probability output, label/tier, and explanation factors.

### Steps
1. Go to Investigation Workspace.
2. In Manual Transaction Scorer, enter a transaction:
   - Sender: `C_TEST_SENDER`
   - Receiver: `C_TEST_RECEIVER`
   - Amount: `850000`
   - Type: `UPI`
3. Click `Score Transaction`.
4. From recent/high-risk transaction list, click `Why?` to open explanation.

### Pass Criteria
1. Score response includes:
   - `fraud_probability` (0-1)
   - `fraud_label`
   - `risk_tier`
2. Explain modal shows top contributing factors (no fallback error).

### Evidence to mention
- Scoring endpoint: `POST /api/score`
- Explain endpoint: `GET /api/explain/{txn_id}`

## Test Case 3: File Upload Mapping + Batch Score Output

### Goal
Show file input handling with schema mapping and model output generation.

### Input file
- Use: `data/raw/demo_upload_sample.csv`

### Steps
1. Go to Investigation Workspace.
2. In Upload Transactions (Column Mapping):
   - Choose `data/raw/demo_upload_sample.csv`
   - Click `Detect Columns`
   - Keep `Score` checked
   - Keep `Persist` unchecked for dry-run (or check it if you want DB insert)
3. Click `Upload`.

### Pass Criteria
1. Upload result shows:
   - `Rows received > 0`
   - `Rows normalized > 0`
   - `Scored: yes`
2. Response includes sample rows with risk fields.

### Evidence to mention
- Upload endpoint: `POST /api/transactions/upload`
- Supports `mapping_json` for non-standard bank CSV schemas.
